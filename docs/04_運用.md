# 運用

本ドキュメントは、ローカル開発環境での定常運用・確認・障害切り分け手順をまとめる。

最終更新: 2026-04-19

---

## 1. 事前準備

1. `.env` を作成し、`POSTGRES_PASSWORD` を必ず設定する
2. API コンテナを起動する

```bash
make up
```

3. ヘルスチェックを確認する

```bash
make health
```

---

## 2. 初期セットアップ

初回構築または環境を作り直したときは以下を実行する。

```bash
make ops-bootstrap
```

`ops-bootstrap` は以下を順に実行する。

- `db-migrate-core` / `db-seed-properties`
- `db-migrate-ops` / `db-migrate-features` / `db-migrate-embeddings` / `db-migrate-learning` / `db-migrate-eval`
- `search-sync`
- `embeddings-generate`
- `training-label-seed` / `training-generate` / `training-fit`

---

## 3. 定常運用

### 日次運用

```bash
make ops-daily
```

実行内容:

- `search-sync`: DB → Meilisearch 同期
- `features-daily`: `property_stats` / `property_features` 日次更新
- `embeddings-generate`: active 物件の embedding 更新
- `kpi-daily`: KPI 日次集計

### 週次運用

```bash
make ops-weekly
```

実行内容:

- `eval-compare`: Meili と rerank の差分確認
- `eval-offline`: オフライン評価を保存
- `eval-weekly-report`: 週次レポート出力と採用判定
- `retrain-weekly`: 採用判定に応じた再学習

## 4. 動作確認

代表確認をまとめて実行する。

```bash

```

内訳:

- `health`

- `feedback-check`
- `ranking-check`
- `ranking-check-verbose`
- `eval-compare`
- `eval-offline`

### pgAdmin 即時確認（Phase 7 着手前）

1. `make up` を実行して `postgres` と `pgadmin` が起動していることを確認
2. ブラウザで `http://localhost:5050` にアクセスし、`.env` の `PGADMIN_DEFAULT_EMAIL` / `PGADMIN_DEFAULT_PASSWORD` でログイン
3. DB 接続を登録
	- Host: `postgres`
	- Port: `5432`
	- DB: `estate`
	- User: `admin`
	- Password: `.env` の `POSTGRES_PASSWORD`
4. 以下のテーブルが参照できることを確認
	- `search_logs`
	- `property_stats`
	- `property_features`
	- `offline_eval_reports`
	- `kpi_daily_stats`
5. 実行直後の状態確認（例）

	- `SELECT * FROM kpi_daily_stats ORDER BY stat_date DESC LIMIT 7;`

## 6. トラブルシュート

### `Search backend unavailable` が返る

1. `make up` を再実行
2. `make logs` で `api` と `meilisearch` のログ確認
3. `MEILI_HTTP_ADDR` と `MEILI_INDEX` の環境変数を確認

### バッチで DB 接続エラー

1. `.env` の `POSTGRES_PASSWORD` 設定を確認
2. `make logs` で `postgres` の起動状態を確認
3. 必要なら `make down && make up` で再起動

### 学習が失敗する

1. `make training-label-seed` で教師データを追加
2. `make training-generate` を実行
3. `make training-fit` を再実行

### 週次再学習がスキップされる

`retrain-weekly` は直近判定が `adopt_lgbm=False` の場合にスキップする。先に `eval-offline` と `eval-weekly-report` の結果を確認する。
