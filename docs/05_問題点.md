# プロジェクト全体 コード問題点整理

作成日: 2026-04-19

検査対象: src/, tests/, requirements*.txt, Dockerfile

## 1. 🔴 エラーハンドリング・耐障害性

### 1.1 API層エラーハンドリング不足

**ファイル**: `src/api/routes/search.py`, `src/api/routes/feedback.py`

**問題**:
- HTTPException のみで catch し、他の予期しない例外がキャッチされない
- Meilisearch/Redis/Database の接続障害時にリトライロジックがない
- 部分的な失敗（例: ログ失敗）時の fallback がない

**具体例**:
```python
# search.py: HTTPError はキャッチするが、他の予期しない例外は伝播
except HTTPError as exc:
    raise HTTPException(status_code=502, detail="Search backend unavailable") from exc
# → TimeoutError, ConnectionError などはキャッチされない
```

**修正提案**:
- グローバル例外ハンドラーを FastAPI に登録
- 接続障害別に異なるステータスコード（502, 504, 503）を返す
- リトライロジックを実装（指数バックオフ）

---

### 1.2 Redis 障害時のサイレント無視

**ファイル**: `src/clients/redis_client.py`

**問題**:
```python
def get_cached_search(params) -> dict | None:
    try:
        raw = _get_client().get(_make_cache_key(params))
        # ...
    except Exception:  # ← 全例外をサイレント無視
        pass
    return None
```

**影響**:
- Redis 接続失敗時にキャッシュ機能が沈黙・無通知
- デバッグが困難
- 本番環境での監視不可

**修正提案**:
- 例外をログに記録（WARNING レベル以上）
- 構造化ログで接続エラーを追跡可能に

---

### 1.3 Meilisearch 接続タイムアウト非対応

**ファイル**: `src/clients/meilisearch_client.py`

**問題**:
- タイムアウト設定は `timeout=10.0` だが、`socket_timeout` の設定がない
- ネットワーク遅延時に無限待機する可能性

**修正提案**:
- `socket_timeout` と `connect_timeout` を明示的に設定

## 4. 🔒 セキュリティ

### 4.1 入力検証の不足

**ファイル**: `src/api/routes/search.py`, `src/api/routes/feedback.py`

**問題**:
- Pydantic モデルがないため、 limit や candidate_limit の制限がクエリパラメータでも信頼されていない
- `q` パラメータの最大長制限がない

**修正提案**:
```python
from pydantic import BaseModel, Field, conint

class SearchParams(BaseModel):
    q: str = Field(default="", max_length=500)
    limit: int = Field(default=20, ge=1, le=100)
    candidate_limit: int = Field(default=100, ge=1, le=200)
```

---

### 4.2 SQL パラメータクエリは安全だが検証が形式的

**ファイル**: `src/repositories/*.py`

**問題**:
- Psycopg の parameterized query は安全だが、**input sanitization** がない
- `apply_feedback()` で `action` が `Literal["click", "favorite", "inquiry"]` に制限されているのは良い

**修正提案**:
- Enum を使用した型安全性強化

## 7. 🐳 Docker・ビルド構成

### 7.1 Dockerfile の最適化不足

**ファイル**: `Dockerfile`

**問題**:
```dockerfile
FROM python:3.11-slim
# → キャッシュ最適化なし
# → マルチステージビルド未使用

COPY src ./src
# → 全体ビルド時に不必要なファイル含まれる可能性
```

**修正提案**:
```dockerfile
# ビルドステージ
FROM python:3.11-slim as builder
RUN pip install --user --no-cache-dir -r requirements.txt

# ランタイムステージ
FROM python:3.11-slim
COPY --from=builder /root/.local /root/.local
ENV PATH=/root/.local/bin:$PATH
```
