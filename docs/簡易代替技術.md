# Elasticsearch 類似・代替・簡易技術リスト

## （Docker対応込み / 不動産検索・LGBM + ME5 前提）

Elasticsearch は強力ですが、

* 学習コスト高い
* JVM重い
* 運用やや複雑
* 個人開発にはオーバースペックな場合あり

そのため、**検索基盤の代替候補** を用途別に整理します。

---

# 結論（おすすめ順）

| 順位 | 技術                         | 用途                | Docker | 難易度 |
| -- | -------------------------- | ----------------- | ------ | --- |
| 1  | Meilisearch                | 最強簡易全文検索          | 〇      | 低   |
| 2  | Typesense                  | Elasticsearch代替本命 | 〇      | 低〜中 |
| 3  | OpenSearch                 | Elastic互換本格派      | 〇      | 中   |
| 4  | PostgreSQL + pg_trgm / FTS | DB内検索             | 〇      | 低   |
| 5  | Qdrant                     | ME5ベクトル検索特化       | 〇      | 低   |
| 6  | Weaviate                   | ハイブリッド検索          | 〇      | 中   |

---

# 🥇 1位 Meilisearch

## 特徴

* 超簡単
* 日本語も工夫すれば可
* API使いやすい
* 軽量高速
* Docker最強クラス

## 向く用途

* 不動産検索PoC
* ポートフォリオ
* 小中規模検索

## Docker

```bash
docker run -p 7700:7700 getmeili/meilisearch
```

## 黒澤様向け評価

**一番楽に成果物になる。**

---

# 🥈 2位 Typesense

## 特徴

* Elasticsearchより簡単
* typo tolerance強い
* facet検索強い
* 不動産検索向き

## Docker

```bash
docker run -p 8108:8108 typesense/typesense:latest
```

## 向く用途

* 価格帯検索
* エリア検索
* 条件絞り込み

---

# 🥉 3位 OpenSearch

## 特徴

* Elasticsearch系互換
* AWS寄り
* 本番案件感強い

## Docker

公式composeあり。

## 向く用途

* SIer案件風
* 企業ポートフォリオ
* Elastic経験アピール

---

# 4位 PostgreSQL + pg_trgm + Full Text Search

## 特徴

* DB一つで完結
* 追加ミドル不要
* Docker簡単

## Docker

```bash
docker run -p 5432:5432 postgres
```

## 向く用途

* 個人開発
* MVP
* 小規模検索

## 強い理由

既にあなたはPostgreSQL強いので即戦力。

---

# 5位 Qdrant

## 特徴

* ベクトル検索専用
* ME5との相性抜群
* 類似物件検索強い

## Docker

```bash
docker run -p 6333:6333 qdrant/qdrant
```

## 向く用途

* 「札幌で静かな感じ」など意味検索
* 類似物件推薦

---

# 6位 Weaviate

* ベクトル + フィルタ
* RAG寄り

あり。
