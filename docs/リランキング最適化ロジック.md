承知しました。
**対象は「ME5で意味類似特徴量を作る」「LightGBM（LGBM）で最終ランキング最適化する」** の2本です。
その前提で、**WSL + Docker でローカル実行**し、その後 **GCP Vertex で学習ジョブ化**する流れを、実装順で整理します。Vertex では、学習コードを自由に載せるなら **custom container** が最も柔軟で、Pipelines で一連の処理をオーケストレーションできます。([Google Cloud Documentation][1])

# 全体像

```text
[WSL + Docker]
  1. 物件データ / ログデータ準備
  2. ME5で embedding生成
  3. LGBM Ranker学習
  4. ローカルで推論確認

[GCP / Vertex]
  5. 学習コードをDocker化
  6. Artifact Registryへpush
  7. Vertex Custom Trainingで学習実行
  8. モデル/特徴量/評価結果をGCSへ保存
  9. 必要ならVertex Pipelines化
```

E5 系の multilingual-e5-large は 1024 次元 embedding を出力するモデルとして公開されており、クエリ・文書の意味類似用途に向いています。LightGBM は ranking 用に `lambdarank` や `rank_xendcg` をサポートしており、`LGBMRanker` が使えます。([Hugging Face][2])

## 2-3. embedding 生成コード例

```python
# app/embed.py
import pandas as pd
import torch
import torch.nn.functional as F
from transformers import AutoTokenizer, AutoModel

MODEL_NAME = "intfloat/multilingual-e5-large"
DEVICE = "cuda" if torch.cuda.is_available() else "cpu"

tokenizer = AutoTokenizer.from_pretrained(MODEL_NAME)
model = AutoModel.from_pretrained(MODEL_NAME).to(DEVICE)
model.eval()

def average_pool(last_hidden_states, attention_mask):
    last_hidden = last_hidden_states.masked_fill(~attention_mask[..., None].bool(), 0.0)
    return last_hidden.sum(dim=1) / attention_mask.sum(dim=1)[..., None]

@torch.no_grad()
def encode_texts(texts, prefix="query: "):
    batch = [prefix + t for t in texts]
    tokens = tokenizer(batch, max_length=512, padding=True, truncation=True, return_tensors="pt").to(DEVICE)
    outputs = model(**tokens)
    embeddings = average_pool(outputs.last_hidden_state, tokens["attention_mask"])
    embeddings = F.normalize(embeddings, p=2, dim=1)
    return embeddings.cpu().numpy()

def main():
    queries = pd.read_csv("data/search_logs.csv")
    props = pd.read_csv("data/properties.csv")

    q_emb = encode_texts(queries["query"].fillna("").tolist(), prefix="query: ")
    p_emb = encode_texts(props["description"].fillna("").tolist(), prefix="passage: ")

    pd.DataFrame(q_emb).to_parquet("artifacts/query_embeddings.parquet")
    pd.DataFrame(p_emb).to_parquet("artifacts/property_embeddings.parquet")

if __name__ == "__main__":
    main()
```

E5 系の使い方では、`query:` と `passage:` のような prefix を付けて encode する例が公開されています。([Hugging Face][2])

# 3. LightGBM でランキング学習

## 3-1. 学習データの形

LGBM Ranker は、**同一検索クエリごとの group 情報** が必要です。
つまり「1検索あたり候補物件N件」という単位でまとめます。LightGBM の ranking API では `group` を使ってクエリ単位のランキングを学習します。([LightGBM][3])

### 例

| query_id | property_id | price | walk_min | area | me5_sim | ctr_hist | label |
| -------- | ----------: | ----: | -------: | ---: | ------: | -------: | ----: |
| q1       |         p11 | 78000 |        5 |   52 |    0.89 |     0.12 |     3 |
| q1       |         p12 | 76000 |       11 |   49 |    0.77 |     0.08 |     1 |
| q1       |         p13 | 85000 |        4 |   61 |    0.82 |     0.15 |     2 |
| q2       |         p21 | 65000 |        8 |   41 |    0.73 |     0.06 |     0 |

`label` は例えば以下でよいです。

* 問い合わせ = 3
* お気に入り = 2
* 詳細閲覧 = 1
* 無反応 = 0

---

## 3-2. 学習コード例

```python
# app/train.py
import os
import joblib
import pandas as pd
from lightgbm import LGBMRanker
from sklearn.model_selection import GroupShuffleSplit
from sklearn.metrics import ndcg_score

DATA_PATH = os.environ.get("TRAIN_DATA", "artifacts/training_pairs.parquet")
MODEL_DIR = os.environ.get("MODEL_DIR", "models")

def main():
    df = pd.read_parquet(DATA_PATH)

    feature_cols = [
        "price", "walk_min", "area", "building_age",
        "photo_count", "favorite_rate", "inquiry_rate",
        "me5_sim_desc", "me5_sim_amenities", "me5_sim_area"
    ]

    X = df[feature_cols]
    y = df["label"]
    groups = df["query_id"]

    splitter = GroupShuffleSplit(n_splits=1, test_size=0.2, random_state=42)
    train_idx, valid_idx = next(splitter.split(X, y, groups=groups))

    train_df = df.iloc[train_idx].copy()
    valid_df = df.iloc[valid_idx].copy()

    X_train = train_df[feature_cols]
    y_train = train_df["label"]
    q_train = train_df.groupby("query_id").size().to_list()

    X_valid = valid_df[feature_cols]
    y_valid = valid_df["label"]
    q_valid = valid_df.groupby("query_id").size().to_list()

    model = LGBMRanker(
        objective="lambdarank",
        metric="ndcg",
        n_estimators=300,
        learning_rate=0.05,
        num_leaves=63,
        min_child_samples=20
    )

    model.fit(
        X_train, y_train,
        group=q_train,
        eval_set=[(X_valid, y_valid)],
        eval_group=[q_valid],
        eval_at=[10]
    )

    os.makedirs(MODEL_DIR, exist_ok=True)
    joblib.dump(model, f"{MODEL_DIR}/lgbm_ranker.joblib")

if __name__ == "__main__":
    main()
```

LightGBM は ranking objective として `lambdarank`、高速な代替として `rank_xendcg` をサポートしています。`rank_xendcg` は `lambdarank` と近い性能で高速と案内されています。([LightGBM][4])

# 9. Vertex Pipelines にする場合

学習を定期化・再現性確保したいなら、次の3段階に分けます。

```text
1. embedding生成コンポーネント
2. feature生成 + 学習データ作成
3. LGBM学習コンポーネント
```

Vertex AI Pipelines は Kubeflow Pipelines / TFX ベースの ML workflow をサーバレスに実行できる仕組みです。([Google Cloud Documentation][6])

### 分ける理由

* embedding 再生成だけやりたい
* 特徴量生成だけ差し替えたい
* 学習だけ再実行したい

この分離ができると実務感が強くなります。

# 12. 実務上の注意

## ME5 側

`multilingual-e5-large` は 1024 次元で比較的重いので、最初はサンプル件数を絞って確認した方が安全です。モデルカードに 1024 次元とあります。([Hugging Face][2])

## LGBM 側

ranking は通常の分類よりも、**query ごとの group 整形ミス** で壊れやすいです。`group` の件数合計が学習行数と一致することを必ず確認してください。LightGBM の ranking API は group ベースです。([LightGBM][3])

## Vertex 側

最初から endpoint デプロイまで行かず、まずは **学習ジョブ完走 + GCS へモデル保存** に絞る方が安全です。Vertex の custom container は training と serving で別設計にもできます。([Google Cloud Documentation][8])
